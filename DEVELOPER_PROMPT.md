# 개발자 전달용 프롬프트

아래 요구사항을 기준으로 B2B 영업 실무용 AI CRM SaaS의 1차 MVP를 개발해 주세요.

## 역할

당신은 SaaS CRM 제품을 개발하는 풀스택 개발자입니다. 목표는 일반적인 고객 관리 도구가 아니라, B2B 솔루션 영업 담당자가 실제 업무에서 사용하는 영업 지원 CRM을 만드는 것입니다.

이번 MVP에서는 전체 CRM을 한 번에 만들지 말고 아래 3개 기능을 우선 구현해 주세요.

1. AI 기반 맞춤형 제안서 및 공식 공문 초안 생성
2. 라이선스 갱신 및 Up-sell/Cross-sell 기회 자동 탐지
3. 과거 Win/Loss 데이터 기반 최적 할인율 및 가격 협상 가이드

## 제품 목표

영업 담당자가 고객사와 영업 건을 선택하면, 해당 고객의 계약 정보, 라이선스 정보, 과거 대응 이력, 유사 Win/Loss 사례를 기반으로 다음 업무를 수행할 수 있어야 합니다.

- 고객 질의에 대한 공식 답변서 또는 제안서 초안 생성
- 라이선스 만료 및 갱신 우선순위 확인
- 업셀/크로스셀 가능 제품 추천
- 견적 제출 전 권장 할인율과 최대 허용 할인율 확인
- 고객 단가 인하 요청에 대한 방어 논리 확인

## 기술 방향

초기 MVP는 실제 AI API가 없어도 동작해야 합니다. 우선 샘플 데이터와 규칙 기반 로직으로 구현하고, 이후 AI API 또는 벡터 검색을 붙일 수 있도록 구조를 분리해 주세요.

권장 구조:

- 프론트엔드: React 또는 Next.js
- 스타일: Tailwind CSS 또는 CSS Modules
- 상태 관리: 초기에는 로컬 상태
- 데이터: 초기에는 mock data 파일
- 추후 확장: API layer, DB, AI service layer 분리 가능하게 설계

실제 백엔드가 없는 경우에도 화면과 사용자 흐름은 완성도 있게 구현해 주세요.

## 기능 2. 라이선스 갱신 및 Up-sell/Cross-sell 탐지

### 화면 구성

- 갱신 D-Day 카드
- Churn 위험도 카드
- 예상 추가 매출 카드
- 자동 탐지 신호 목록
- 추천 액션 목록
- 추천 제안 메시지

### 계산 기준

현재 기준일은 개발 편의를 위해 `2026-05-28`로 고정해도 됩니다.

갱신 D-Day:

- 라이선스 만료일 - 현재 기준일

Churn 위험도 예시:

- 기본 위험도 = 100 - 고객 건강도
- 만료일이 90일 이내이면 +10
- 사용량이 85% 이상이면 +5
- 최근 불만 또는 실패 이력이 있으면 +10

업셀 추천 조건:

- OTP만 사용 중이고 SSO 미도입이면 SSO 연동 추천
- 사용량이 85% 이상이면 라이선스 증설 추천
- DR 또는 서버 이중화 미도입이면 이중화/DR 구성 추천
- VPN 인증만 사용 중이면 내부 업무 포털 인증 확대 추천

예상 추가 매출:

- 기존 예상 매출의 20~35% 범위로 계산
- 사용량 85% 이상이면 높은 비율 적용

### 추천 액션 예시

- 갱신 제안서 작성
- 기술 점검 미팅 제안
- SSO 확장 제안
- 라이선스 증설 제안
- 유지보수 3개월 무상 제공 대신 할인율 방어

추천 로직은 `detectRenewalOpportunities(customer)` 같은 함수로 분리해 주세요.

## 기능 3. Win/Loss 기반 가격 가이드

### 화면 구성

- 사용자 수 입력
- 요청 할인율 입력
- 경쟁사 참여 여부
- 서버 이중화 포함 여부
- 권장 할인율 표시
- 최대 허용 할인율 표시
- 예상 수주 확률 표시
- 마진 위험 경고
- 유사 Win/Loss 사례 테이블
- 가격 방어 논리 표시

### 계산 기준

권장 할인율:

- 동일 산업군 Win 사례 평균 할인율
- 경쟁사 참여 시 +2%
- 사용자 수가 3,000 이상이면 +1%

최대 허용 할인율:

- 유사 Loss 사례 평균 할인율보다 1~2% 낮게 설정
- 단, 25%를 넘지 않도록 제한

예상 수주 확률:

- 기본값 70%
- 요청 할인율이 권장 할인율과 가까우면 상승
- 요청 할인율이 최대 허용 할인율을 초과하면 큰 폭으로 하락
- 경쟁사 참여 시 소폭 하락

마진 경고:

- 요청 할인율이 최대 허용 할인율 이상이면 경고
- 권장 할인율보다 높으면 대안 조건 제시

가격 방어 논리:

- 추가 할인 대신 무상 유지보수 3개월
- 구축 지원 범위 조정
- 서버 이중화 옵션 별도 견적
- 단계적 사용자 증설 계약

가격 로직은 `calculatePricingGuide(deal, cases, input)` 같은 함수로 분리해 주세요.

## 개발 제외 범위

이번 MVP에서는 아래 기능을 구현하지 않아도 됩니다.

- 실제 로그인/권한
- 실제 DB 저장
- 실제 AI API 호출
- 이메일 계정 연동
- PDF 다운로드
- 전자결재
- ERP/회계 연동
- 모바일 앱

단, 나중에 붙일 수 있도록 컴포넌트와 서비스 함수를 분리해 주세요.

## 완료 기준

아래 항목을 만족하면 1차 MVP 완료로 봅니다.

- 고객사를 변경하면 세 기능의 결과가 모두 바뀐다.
- 문서 생성 화면에서 CRM 데이터 기반 초안이 생성된다.
- 갱신/업셀 화면에서 D-Day, 위험도, 추천 액션이 표시된다.
- 가격 가이드 화면에서 권장 할인율, 마지노선, 수주 확률, 방어 논리가 표시된다.
- 유사 Win/Loss 사례가 화면에 표시된다.
- 주요 로직은 UI 안에 직접 박지 말고 별도 함수로 분리되어 있다.
- mock data를 실제 API 데이터로 교체하기 쉬운 구조다.

## 중요 디자인 기준

이 제품은 실무용 CRM이므로 과한 랜딩페이지나 마케팅형 디자인은 피해주세요. 첫 화면부터 실제 업무 화면이어야 합니다.

원하는 분위기:

- 차분한 SaaS 운영 도구
- 정보 밀도가 높지만 복잡하지 않은 화면
- 고객/딜/추천 근거가 빠르게 스캔되는 레이아웃
- 영업 담당자가 매일 켜놓고 쓰는 업무형 UI

피해야 할 것:

- 의미 없는 히어로 섹션
- 장식 위주의 카드 남발
- 실제 데이터 없이 설명문만 많은 화면
- AI 기능을 과장하는 문구
- CRM보다 랜딩페이지처럼 보이는 구성
